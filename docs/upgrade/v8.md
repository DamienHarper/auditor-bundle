# Upgrading to 8.0

> **Migrate from auditor-bundle 7.x to 8.0**

This guide covers upgrading from auditor-bundle 7.x to 8.0.

## ⚠️ DoctrineProvider extraction completed in auditor 5.0

`auditor-bundle` 8.0 targets `damienharper/auditor` 5.x, in which the embedded
`DH\Auditor\Provider\Doctrine` namespace has been **fully removed** from the core library.

All Doctrine ORM functionality now lives exclusively in
[`damienharper/auditor-doctrine-provider`](https://github.com/DamienHarper/auditor-doctrine-provider),
which auditor-bundle already requires as a direct dependency.

> [!IMPORTANT]
> The class namespace **has not changed**. Classes are still under `DH\Auditor\Provider\Doctrine\*`
> — they simply come from the `auditor-doctrine-provider` package instead of the `auditor` core.
> No import statements need to be updated.

### For auditor-bundle users

No action required. The bundle automatically requires `damienharper/auditor-doctrine-provider`
and wires everything transparently. Your `dh_auditor.yaml` configuration continues to work
without any changes.

### For users using DoctrineProvider directly (without the bundle)

If you use `DoctrineProvider` directly in a non-bundle project, install the standalone package
explicitly:

```bash
composer require damienharper/auditor-doctrine-provider:^2.0
```

All imports remain identical:

```php
// Works in both 4.x (from auditor core) and 5.x (from auditor-doctrine-provider)
use DH\Auditor\Provider\Doctrine\DoctrineProvider;
use DH\Auditor\Provider\Doctrine\Configuration;
use DH\Auditor\Provider\Doctrine\Persistence\Reader\Reader;
// ...
```

### `NeedsConversionToAuditableType` (if applicable)

If you implemented the `NeedsConversionToAuditableType` interface on a custom Doctrine type,
update the import:

```php
// Before (auditor 4.x)
use DH\Auditor\Transaction\NeedsConversionToAuditableType;

// After (auditor 5.x / auditor-doctrine-provider 2.x)
use DH\Auditor\Provider\Doctrine\Transaction\NeedsConversionToAuditableType;
```

## 🚀 Upgrade Steps

### 1️⃣ Update auditor-bundle

```bash
composer require damienharper/auditor-bundle:^8.0
```

This will automatically pull in `damienharper/auditor:^5.0` and
`damienharper/auditor-doctrine-provider:^2.0`.

### 2️⃣ Clear Cache

```bash
bin/console cache:clear
```

### 3️⃣ Verify Schema

```bash
bin/console audit:schema:update --dump-sql
```
